# Claude Code — project context

This repository uses **AGENTS.md** as the canonical description of architecture, conventions, and pitfalls.

## Before making changes

1. Read `AGENTS.md`.
2. Follow `CONTRIBUTING.md` to make contributions to the codebase
3. Follow `REDHAT.md` for Red Hat AI policy (attribution, sensitive data, upstream).
4. Prefer small, reviewable steps; ask for constraints (ex. single skills, dependencies).

## Working agreements

- Propose a short plan for non-trivial work; then implement.
- After edits, suggest **tests** or **verification** commands from the project (or ask the user what they use).
- For commits: remind the user to add `Assisted-by:` or `Generated-by:` when appropriate.
